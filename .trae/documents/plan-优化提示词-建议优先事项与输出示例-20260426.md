# /plan：优化结构化分析提示词（建议优先事项 + 输出示例澄清）

## 目标

1. “建议优先处理事项”要基于合同**已存在缺陷**与**未提及/缺失风险**的综合优先级输出（而不是只列其中一类）。
2. “输出格式”作为**样例**说明，实际输出应根据合同内容与分析结果生成；同时保持后端 JSON 解析稳定。

## 现状（已核对）

- 结构化分析提示词在 [AiContractAnalyst.analyzeContract](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java) 的 `@SystemMessage` 中硬编码。
- 后端解析对缺失字段具备容错（`risk_level` 默认“中”，数组字段默认 `[]`），但仍建议 AI 始终输出固定 key 以避免不稳定。
- “建议优先处理事项”对应 JSON 的 `suggestions` 字段，初次分析回答会将其拼成“建议优先处理事项”段落。

## 改动方案

### A. 强化“建议优先处理事项”的定义与排序规则

在 `【分析要求】` 增加约束：

- `suggestions` 必须是“按优先级从高到低排序”的清单；
- 优先级综合：风险等级/影响范围/发生概率/修复成本/是否导致重大争议（同时考虑缺失项与已存在条款）；
- 每条建议应是可落地的动作（能直接改合同），必要时标注“【缺失】/【现有条款】”。

### B. 将“输出格式”明确为样例，并要求固定 key + 允许空数组

将当前“格式如下”改为“示例（仅示例）”，并补充说明：

- 实际输出须根据合同生成，不要照抄示例内容；
- 仍必须输出同一套 key（允许 `party_*_risks/suggestions/contract_tags` 为空数组）。

## 涉及文件

- 修改：[AiContractAnalyst.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java)

## 验收方式

- 初次分析输出的 `suggestions`：
  - 不再出现无意义占位；
  - 有明确排序（第一条通常对应最高风险或最关键缺失项）；
  - 能同时覆盖“现有条款缺陷”和“缺失条款风险”的优先修复建议。
- JSON 仍可被后端解析并正常落库与前端展示。
