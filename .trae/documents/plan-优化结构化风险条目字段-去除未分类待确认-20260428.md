# /plan：优化结构化分析的风险条目字段（去除“未分类风险/待确认”占位）

## Summary
当前初次分析的展示文本中，风险条目标题经常出现“未分类风险（待确认）”，但条款/说明/法律依据/建议等正文是有内容的。这说明结构化结果里每条风险对象缺少或为空：
- `risk_type`（用于标题）
- `risk_level`（用于括号内的等级）

目标：从源头让结构化 JSON 的 `party_lessor_risks / party_tenant_risks` **每条都输出非空的 risk_type 与 risk_level（高/中/低）**，从而前端展示不再依赖默认占位文案。

## Current State Analysis（基于仓库已核对）
- 展示占位发生位置：后端拼接初次分析答案时使用默认值：
  - `risk_type` 缺失 → “未分类风险”
  - `risk_level` 缺失 → “待确认”
  - 代码：[AnalysisService.appendRiskSection](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L489-L507)
- 结构化输出由提示词约束，但目前并未强制“字段必须非空”，模型可能输出空字符串或省略字段：
  - 代码：[AiContractAnalyst.analyzeContract / analyzeContractConcise](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java)
- 当前后端对 risks 的 schema 做了“必须为对象数组”的严格检查，但没有对对象内部必填字段做校验，因此会出现“对象存在但字段为空/缺失”的情况。
  - 代码：[AnalysisService.readObjectArrayJsonStrict](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java)

## Proposed Changes

### 1) 强化提示词：risk_type / risk_level 必须非空且符合枚举
文件：
- [AiContractAnalyst.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java)

改动点：
- 在“数组要求/硬性约束”增加：
  - 每个风险对象必须包含 `risk_type` 且为 2–12 字的短标签（例：押金返还、违约金畸高、抵押风险、维修报销、备案义务、隐私入户、占用费过高等）
  - 每个风险对象必须包含 `risk_level` 且只能是 高/中/低（不得为空，不得输出“中风险/中高风险”等）
- 增加“自检要求”：输出前检查每个风险对象的字段是否齐全且非空。

### 2) 后端增加“必填字段校验”：不满足则触发一次重试
文件：
- [AnalysisService.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java)

改动点：
- 在结构化解析落库前，对 `party_lessor_risks / party_tenant_risks` 的每个对象校验：
  - `risk_type` 非空
  - `risk_level` ∈ {高, 中, 低}
  - `clause_text` / `risk_description` / `suggestion` 至少非空（避免空壳对象）
- 若不满足，抛出 `JsonProcessingException`（或自定义异常并在 `generateAndSaveStructuredResultWithRetry` 捕获），触发 `analyzeContractConcise` 重试一次。
- 重试后仍不满足：按现有逻辑返回 SSE error（保持“正常工作”的失败闭环）。

### 3) 展示层降噪（可选，保持最小变更）
文件：
- [AnalysisService.appendRiskSection](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L489-L507)

改动点（可选）：
- 当 `risk_type` 为空但 `clause_text` 存在时，用 clause_text 的前若干字作为标题备选，减少“未分类风险”出现概率。
- 说明：这属于展示兜底，不改变结构化质量；若你坚持“完全从源头解决”，此项可以不做。

## Assumptions & Decisions
- 你希望“从源头解决”，因此核心是提示词强约束 + 后端必填校验触发重试；展示层兜底仅作为可选项。
- 不新增/修改数据库字段与前端结构，保持兼容。

  - 若模型输出缺字段，会触发一次重试；仍失败则 SSE error，且不会落库不完整结构化结果
- 静态检查：IDE diagnostics 无新增错误。
