# /plan：提示词增强——“未提及/缺失的风险也要提示”

目标：让 AI 在分析时不仅指出合同中“已写明的风险条款”，也要指出“未约定/未提及但应当约定的关键事项”带来的风险，并给出可落地的补充条款建议与法律依据；同时保持现有前后端数据结构不变、解析稳定。

范围（本次要改的点）

- 结构化分析（初次分析 JSON 输出）：增强提示词，强制覆盖“缺失条款风险”。
- 追问（非流式与流式两套 Prompt）：增强提示词，让回答在聚焦追问的同时，提醒相关的关键缺失风险。

不做的点（本次不改）

- 不新增数据库字段与后端 JSON 解析字段（避免回归面扩大）。
- 不改前端展示组件的数据结构与渲染逻辑。

## 方案设计（不改数据结构，靠约定编码缺失项）

### A. 结构化分析（JSON）增加“缺失条款风险”的硬性要求

在【分析要求】增加 2 类要求：

1. 必须识别“合同未约定/未提及但对租房重要的事项缺失”带来的风险（例如：押金退还时限、维修响应、提前解约条件、交接清单、发票/收据、水电物业结算、房屋权属与可出租证明、不可抗力/房屋被查封等）。
2. 对缺失项必须给出“建议补充条款文本（可直接粘贴到合同里）+ 法律依据 + 风险等级”。

### B. 缺失项如何放进现有 JSON（避免改解析/存储）

约定缺失项仍然写入 `party_tenant_risks` / `party_lessor_risks` 数组中：

- `clause_index`: 使用 `0`
- `clause_text`: 以 `【缺失】` 开头，例如：
  - `【缺失】押金退还期限与扣减规则（建议补充：…）`
- `risk_type/risk_level/risk_description/legal_basis/suggestion` 正常填写

这样后端现有解析与前端条款卡片展示无需改动即可展示“缺失项风险”。

### C. 追问 Prompt（流式与非流式）增加“补充提醒”规则

在【回答要求】中加入：

- 在回答用户追问后，若发现与追问主题强相关的“缺失条款风险/未提及风险”，必须追加一个“补充提醒”小段落（不强行扩散到不相关主题）。
