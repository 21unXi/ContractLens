# /plan：修复结构化分析稳定性（强约束 JSON + 风险数组兼容，减少“未提供/待确认”）

## Summary
当前初次分析在结构化阶段存在两类问题：
1) 模型返回的 JSON 偶发被截断，导致解析失败（`JsonEOFException`）。
2) 即使解析成功，`party_*_risks` 可能是字符串数组而不是对象数组，导致回答拼接时出现“未分类风险（待确认）/未提供”占位。

目标：让“建立会话 → 检索 → 结构化解析 → 展示”稳定工作；不依赖“最小可用结果”兜底来掩盖问题，而是通过**强约束输出**与**输入兼容转换**保证可展示的数据结构一致。

  - [AiContractAnalyst.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java)
  - [AnalysisService.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L359-L413)
- 初次分析的“文字答案”由 `buildInitialAnswer` 拼接，其中 `appendRiskSection` 假设 risks 数组元素是对象；当数组元素是字符串时会出现占位：
  - [AnalysisService.appendRiskSection](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L489-L507)
- 配置已设置 `langchain4j.open-ai.chat-model.response-format=json_object` 等，但模型仍可能输出不完整 JSON（通常与输出过长/服务端截断/模型未遵循指令有关）。
  - [application.yml](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/resources/application.yml#L28-L38)

## Proposed Changes

### 1) 去除“最小可用结构化结果”降级路径，改为“强约束 + 兼容转换”
文件：
- [AiContractAnalyst.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java)
- [AnalysisService.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java)

做法：
- 保留一次重试（同一 schema、更短输出），但删除第三次 `analyzeContractMinimal` 的降级调用，避免生成低质量结构化结果后继续向下游传播。
- 解析成功后，增加 schema 规整：
  - 将 `party_lessor_risks / party_tenant_risks` 的“字符串数组”自动转换成“对象数组”，用于前端正常展示（不出现“未提供”）。
  - 将 `risk_level` 归一化为：高/中/低（例如把“中高风险”归一为“中”）。

### 2) 限制上下文体量，降低输出过长与截断概率
文件：[AnalysisService.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java)

做法：
- 对 `retrieved_context` 做总长度限制（例如限制总字符数），避免“合同原文 + 大段检索上下文 + 图谱上下文”导致模型更容易跑长并截断。
- 规则：优先保留 topK 片段，但每段截断到固定长度，总长度达到上限即停止拼接。

### 3) 优化提示词的“结构完整性”强调（不增加字段，仅强化约束）
文件：[AiContractAnalyst.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java)

做法：
- 增加“必须闭合 JSON、数组/对象必须完整”的硬性约束。
- 重申条目上限（每视角最多 5 条）+ 字段长度上限（`clause_text` 等），减少输出体量。

### 4) 失败时明确报错（不输出占位文本）
文件：[AnalysisService.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java)

做法：
- 若两次生成都无法解析：保持失败（SSE error），并返回简短可操作提示（例如建议重试、调小输出规模、提高 timeout）。
- 避免进入“解析成功但 schema 不对”导致的占位展示。

## Assumptions & Decisions
- 不能保证任何第三方模型“永远输出合法且完整的 JSON”。能做的是：限制输出体量、强约束、失败重试、解析后规整、失败明确可重试。
- 用户偏好：当 risks 为字符串数组时允许自动转换为对象数组（用于正常展示）。

## Verification Steps
- 用同一份合同重复触发初次分析：
  - `party_*_risks` 在 UI 中不再出现“未分类风险（待确认）/未提供”占位。
  - 若模型仍输出截断 JSON：最多重试一次，然后返回清晰错误，不输出低质量“最小结果”。
- 静态检查：IDE diagnostics 无新增错误。
