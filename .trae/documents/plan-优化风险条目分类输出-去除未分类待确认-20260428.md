## Summary
- 目标：让“租客/房东视角重点风险”每条风险都输出明确的风险类型与风险等级，不再出现“未分类风险（待确认）”这种占位标题。
- 核心策略：从源头约束 LLM 输出 + 后端严格校验（含禁止占位值）+ 任何不合格结构化结果直接触发重试/报错，而不是生成占位文本答案。

## Current State Analysis
- 当前“未分类风险（待确认）”来源于首轮答案拼接的兜底：
  - [AnalysisService.appendRiskSection](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L476-L494) 对 `risk_type/risk_level` 为空时分别兜底为“未分类风险/待确认”。
  - [AnalysisService.buildInitialAnswer](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L462-L474) 对总体 `riskLevel` 为空时兜底为“待确认”。
- 结构化 JSON 入库前已做“对象数组 + 必填字段 + risk_level 归一化”校验：
  - [AnalysisService.mapJsonToAnalysisResult](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L391-L421)
  - [AnalysisService.readRiskArrayJsonStrict](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L781-L825)
- 但仍可能出现两类“看似不空、实为占位”的情况，导致答案里标题没有信息密度：
  1) LLM 输出 `risk_type="未分类风险"`、`risk_level="待确认/未知"` 等占位值（当前校验不拦截）。
  2) LLM 输出 `risk_level` 缺失/空字符串时，会被 [normalizeRiskLevel](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L735-L750) 默认压到“中”，掩盖缺失，降低“失败即重试”的确定性。

## Proposed Changes

### 1) 后端：对风险条目做“反占位”与“缺失即失败”校验
**文件**：`contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java`
- 调整 `readRiskArrayJsonStrict`：
  - 对 `risk_level`：先读取原始 `risk_level`，若为空/缺失直接抛 `JsonProcessingException`；再做 `normalizeRiskLevel` 并写回。
  - 对 `risk_type`：新增“禁用占位值”校验（例如：`未分类风险/未分类/其他/待确认/未知`，以及仅由标点/空白组成），命中则抛异常触发重试。
  - 可选（建议）：对 `risk_type` 长度做约束（例如 2–20 个汉字/字符），过长视为无效以避免把一整段描述塞进标题字段。
- 调整 `appendRiskSection`：
  - 移除对 `risk_type/risk_level` 的“未分类风险/待确认”兜底，改为：字段缺失则跳过该条风险（或直接不拼接该 section）。
  - 依赖上游严格校验保证正常情况下不会缺字段；这样即使历史数据存在缺字段，也不会再输出误导性的占位标题。

### 2) 提示词：明确禁止占位词，并要求短标签化 risk_type
**文件**：`contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java`
- 在系统提示中补充硬约束：
  - `risk_type` 必须是“短标签”（例如 4–12 字，偏分类命名），禁止输出“未分类风险/待确认/未知/其它”等占位词。
  - `risk_level` 只能是 `高/中/低` 三选一，禁止“待确认/中风险/中高”等（如出现必须自行改写为三选一）。
  - 输出前自检：逐项检查 risks 数组中每个对象字段非空且满足枚举约束。
- `analyzeContractConcise` 的提示同样同步上述约束，保证重试路径一致。

### 3)（可选）前端：不做占位兜底，仅确保空值不破 UI
**文件**：`contractlens-frontend/src/components/ClauseCard.vue`
- 当前直接展示字段，不兜底；保持不改动。
- 若需要，可在展示层对空值做最小容错（例如空字符串显示“（缺少风险类型）”），但默认不做以推动后端从源头保证数据完整。

### 4) 进度记录
**文件**：`PROGRESS.md`
- 按项目规则：本次实现完成后补充“风险条目字段强约束 + 禁止占位 + 重试触发条件强化”的已完成记录，并更新后续建议（如需要）。

## Assumptions & Decisions
- 决策：系统不再在首轮答案中输出“未分类风险（待确认）”这类占位标题；遇到缺字段/占位字段应触发一次重试，仍失败则返回可重试错误（SSE `error`），而不是拼接占位内容。
- 决策：`risk_type` 以 LLM 生成的短标签为主，不引入固定枚举表（避免过度约束导致模型输出崩坏）；但会禁止明显占位词并限制长度。
- 假设：当前结构化入库路径始终走 `readRiskArrayJsonStrict`（已在 [mapJsonToAnalysisResult](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L391-L421) 中确认）。
