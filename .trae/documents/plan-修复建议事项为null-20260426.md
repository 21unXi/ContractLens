# /plan：修复“建议优先处理事项”输出为 null

## Summary
当前初次分析的“建议优先处理事项”出现 `1. null` 这类输出，是因为 AI 返回的 `suggestions` 数组里包含 `null` 元素，后端在拼接初次分析回答时直接把 `null` 追加进 `StringBuilder`，最终被渲染为文本 `null`。本计划将：
- 让展示层不再输出 `null`，当建议项全部无效时输出“暂无建议优先处理事项”；
- 让落库层在写入 `analysis_results.suggestions` 时清洗掉 `null/空字符串`；
- 在提示词中明确禁止输出 `null` 作为占位。

## Current State Analysis (grounded)
- 初次分析的“建议优先处理事项”来自后端拼接的初次分析回答 `buildInitialAnswer(...)` → `appendSuggestionsSection(...)`。
- 当前实现对 `suggestions` 的每个元素：
  - 若是文本：直接 `asText()`；
  - 否则尝试读 `suggested_text/reason`，两者都为空时 `StringBuilder.append(null)` → 输出 `null`。
  - 代码位置：[AnalysisService.appendSuggestionsSection](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L407-L429)
- `suggestions` 的落库值来自 AI JSON：`result.setSuggestions(readJson(root, "suggestions", "[]"))`，目前不做清洗，因此 `[null,null,...]` 会被原样保存。
  - 代码位置：[AnalysisService.mapJsonToAnalysisResult](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java#L245-L274)

## Proposed Changes

### 1) 展示层：过滤无效建议并在为空时输出“暂无”
文件：[AnalysisService.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java)
- 修改 `appendSuggestionsSection(...)`：
  - 跳过 `suggestionNode.isNull()/isMissingNode()`；
  - 对文本节点：取 `trim()` 后为空则跳过；
  - 对对象节点：尝试读取 `suggested_text`/`reason`/（可选再加 `suggestion` 作为兜底），拼成文本后为空则跳过；
  - 只对有效项编号（编号连续）。
  - 若过滤后没有有效项：
    - 输出：
      - `建议优先处理事项：`
      - `暂无建议优先处理事项`

### 2) 落库层：清洗 suggestions，避免把 null 写进 analysis_results
文件：[AnalysisService.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/AnalysisService.java)
- 在 `mapJsonToAnalysisResult(...)` 写入 `result.setSuggestions(...)` 前增加清洗：
  - 新增一个私有方法（例如 `sanitizeStringArrayJson(root, "suggestions")`），只保留非空字符串元素；
  - 若 `suggestions` 不是数组或全无效，则写入 `"[]"`；
  - 这样即使后续展示逻辑有疏漏，也不会再产生 `null` 文本。

### 3) 提示词：禁止输出 null 占位
文件：[AiContractAnalyst.java](file:///e:/code/repository/ContractLens/contractlens-backend/src/main/java/com/contractlens/service/ai/AiContractAnalyst.java)
- 在结构化分析 `analyzeContract` 的输出要求里补充：
  - `suggestions` 必须是“非空字符串数组”，不得包含 `null`，不得用 `null` 作为占位；如无建议请输出空数组 `[]`。

## Assumptions & Decisions
- 决策：当过滤后没有有效建议时，按你的选择输出“暂无建议优先处理事项”（而不是完全不展示）。
- 兼容性：无需数据库迁移；对历史数据（已存 `[null,...]`）也会在展示层被过滤并显示“暂无”。

## Verification
- 构造一个旧数据场景：`suggestions` 为 `[null,null]` 或 `[{...字段缺失...}]`：
  - 初次分析拼接回答中不再出现 `1. null`；
  - 该段展示为“暂无建议优先处理事项”。
- 新分析产生的 `analysis_results.suggestions` 不再包含 `null`（应为 `[]` 或有效字符串数组）。
- 检查后端/前端诊断无新增编译错误（IDE diagnostics）。
